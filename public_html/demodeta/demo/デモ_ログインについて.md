# デモ画面でのログインについて

## このフォルダ（demodeta/demo/）について

- **ビルド成果物のみ**が置かれています。**React や Vite のソースコードはこのリポジトリには含まれていません。**
- デモの元アプリは別の場所でビルドされ、その出力がここに配置されている想定です。ソースの場所が分かれば、そこで「デモ時はログイン不要」を実装してから再ビルドし、ここを上書きしてください。

## 実施済み（このリポジトリ内で可能な範囲）

- **Google GSI スクリプトの削除**: `index.html` から `accounts.google.com/gsi/client` の読み込みを外しました。デモページでは Google アカウント用のスクリプトは読み込まれません。

## ログインを完全に外す場合（ソース側の対応）

デモ用の JS（`assets/*.js`）には Firebase Auth が組み込まれており、**「デモではログイン不要でそのまま使える」**ようにするには、**アプリのソース側**で次のいずれかが必要です。

1. **Firebase 匿名認証を使う**  
   デモ用エントリやデモ用ビルドで、起動時に `signInAnonymously()` を実行し、ログイン画面を出さない。
2. **デモモードを追加する**  
   URL（例: パスに `demodeta/demo` が含まれる）や環境変数（例: `VITE_DEMO=1`）でデモモードを判定し、認証をスキップしてローカル／ダミーデータだけで動作させる。
3. **デモ用ビルドを分ける**  
   本番用とデモ用でビルドを分け、デモ用では上記 1 または 2 を有効にしたうえで、そのビルド成果物をこの `demodeta/demo/` に配置する。

ソースが別リポジトリや別フォルダにある場合は、そこで上記を実装してから再度ビルドし、生成された `index.html` と `assets/*` をこの `demodeta/demo/` に上書きしてください。  
**demodeta の構成・ソースの有無については、同階層の `README.md` も参照してください。**
