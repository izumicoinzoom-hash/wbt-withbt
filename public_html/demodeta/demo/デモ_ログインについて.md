# デモ画面でのログインについて

## 実施済み
- **Google GSI スクリプトの削除**: `index.html` から `accounts.google.com/gsi/client` の読み込みを外しました。デモページでは Google アカウント用のスクリプトは読み込まれません。

## ログインを完全に外す場合
デモ用のビルド（`demodeta/demo/` の JS）は、Firebase Auth による認証が組み込まれた状態でビルドされています。**「デモではログイン不要でそのまま使える」**ようにするには、アプリの**ソース側**で次のいずれかの対応が必要です。

1. **Firebase 匿名認証を使う**  
   デモ用のエントリやビルドで、起動時に `signInAnonymously()` を実行し、ログイン画面を表示しない。
2. **デモモードを追加する**  
   URL や環境変数で「デモモード」を判定し、認証をスキップしてローカル／ダミーデータだけで動作させる。
3. **デモ用ビルドを分ける**  
   本番用とデモ用でビルドを分け、デモ用では上記 1 または 2 を有効にしたうえで、そのビルド成果物を `demodeta/demo/` に配置する。

ソースコードが別リポジトリや別フォルダにある場合は、そこで上記を実装したうえで再度ビルドし、生成された `assets/*.js` などをこの `demodeta/demo/` に上書きしてください。
