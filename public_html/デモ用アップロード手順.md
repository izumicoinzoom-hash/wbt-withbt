# デモ画面を表示するためのアップロード手順（2点）

withbt.com で **https://withbt.com/kiyota/demo/** を開いたときにデモ画面が出るようにするため、次の **2点** をサーバーに反映してください。

---

## 1. ルートの .htaccess をサーバーに反映する

**やること:** WordPress が `/kiyota/demo/` を奪わないようにした修正済み `.htaccess` を、サーバーのルートに上書きアップロードする。

| 項目 | 内容 |
|------|------|
| **ローカル（アップロード元）** | `WBT\public_html\.htaccess` |
| **サーバー上の配置先** | `public_html/.htaccess`（サイトの公開フォルダの直下） |
| **操作** | 上記ローカルファイルを、サーバー側の `public_html/.htaccess` に**上書き**でアップロードする |

※ これで `/kiyota/demo` へのアクセスが WordPress に渡らず、実ファイルが表示されます。

---

## 2. デモ用フォルダ一式をサーバーにアップロードする

**やること:** デモ用の HTML・JS・CSS などを、サーバーの `public_html/kiyota/demo/` に配置する。

| 項目 | 内容 |
|------|------|
| **ローカル（アップロード元）** | `WBT\public_html\kiyota\demo\` **フォルダの中身一式** |
| **サーバー上の配置先** | `public_html/kiyota/demo/` |

**アップロードするファイル一覧（すべて `kiyota/demo/` 配下に同じ階層で）：**

- `index.html`
- `.htaccess`
- `assets/index-2ZP8qj_P.js`
- `assets/index-bA-zNoRr.css`
- `estimator/見積もりチェッカー2.html`

**操作のコツ:**

1. FTP（FileZilla など）でサーバーに接続する。
2. サーバー側で **`public_html/kiyota/`** を開く。
3. **`demo`** フォルダがなければ作成する。
4. ローカルで **`WBT\public_html\kiyota\demo\`** を開き、**中身（index.html や assets フォルダなど）をすべて選択**して、サーバー側の **`public_html/kiyota/demo/`** にドラッグ＆ドロップする。
5. 既存ファイルがある場合は「上書き」でOK。

---

## アップロード後の確認

1. ブラウザで **https://withbt.com/kiyota/demo/** を開く（末尾の `/` を付ける）。
2. キャッシュを無視して再読み込みする（Ctrl+Shift+R または Cmd+Shift+R）。
3. デモ用のログイン画面 → デモバナー付きの工程管理画面が表示されればOKです。
